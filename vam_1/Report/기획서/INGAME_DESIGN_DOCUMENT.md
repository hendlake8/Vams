## 1. 게임 개요

### 1.1 장르 정의
- **메인 장르**: 뱀서라이크 (Vampire Survivors-like)
- **서브 장르**: 로그라이크 슈팅, 캐주얼 액션

### 1.2 핵심 컨셉
| 항목 | 설명 |
|------|------|
| 타겟 유저 | 캐주얼 모바일 게이머 |
| 세션 타임 | 3~5분/판 |
| 조작 방식 | 한 손 조작 (이동만 조작, 공격 자동) |
| 그래픽 | 2D 탑다운 뷰 |

### 1.3 플랫폼 및 화면 사양

#### 1.3.1 타겟 플랫폼
| 항목 | 사양 |
|------|------|
| 플랫폼 | Android |
| 최소 SDK | API 21 (Android 5.0 Lollipop) |
| 타겟 SDK | API 34 (Android 14) |

#### 1.3.2 해상도 정책

| 항목 | 값 | 비고 |
|------|-----|------|
| 기준 해상도 | 1080 x 1920 | 세로 모드 (Portrait) |
| 종횡비 | 9:16 | 기준 종횡비 |
| 화면 모드 | **풀스크린** | 레터박스 사용 안 함 |

#### 1.3.3 해상도 대응 전략


#### 1.3.4 SafeArea 대응

| 요소 | 대응 방식 | 설명 |
|------|-----------|------|
| **노치(Notch)** | 완벽 회피 | 상단 UI 요소 노치 영역 침범 금지 |
| **펀치홀(Punch-hole)** | 완벽 회피 | 카메라 영역 UI 배치 금지 |
| **라운드 코너** | 마진 적용 | 모서리 UI 요소 8px 이상 마진 |
| **네비게이션 바** | 동적 대응 | 제스처/버튼 모드 자동 감지 |
| **상태 바** | 숨김 처리 | 풀스크린 몰입 모드 적용 |


## 2. 코어 게임플레이

### 2.1 조작 시스템

#### 2.1.1 이동
| 요소 | 상세 |
|------|------|
| 입력 방식 | 가상 조이스틱 (화면 좌측 하단) |
| 조이스틱 크기 | 반지름 80px (조절 가능) |
| 데드존 | 10px |
| 이동 속도 | 기본 5.0 unit/s |
| 가속/감속 | 즉시 반응 (0.1s 이내) |

#### 2.1.2 공격
| 요소 | 상세 |
|------|------|
| 공격 방식 | 완전 자동 |
| 타겟팅 | 무기별 상이 (근접 우선, 범위, 임의) |
| 조준 방향 | 캐릭터 시선 방향 (일부 무기) |

### 2.2 전투 시스템

#### 2.2.1 데미지 계산 공식
```
최종 데미지 = (기본 공격력 × 무기 계수 × 스킬 계수) × (1 + 크리티컬 배율) × 속성 보너스

기본 공격력 = 캐릭터 기본 ATK + 장비 ATK 합계 + 버프 ATK
무기 계수 = 무기별 고정 배율 (0.5 ~ 3.0)
스킬 계수 = 스킬 레벨별 배율 (1.0 ~ 2.5)
크리티컬 배율 = 크리티컬 발동 시 추가 배율 (기본 1.5, 최대 3.0)
속성 보너스 = 속성 상성에 따른 배율 (0.8 ~ 1.5)
```

#### 2.2.2 크리티컬 시스템
| 스탯 | 기본값 | 최대값 |
|------|--------|--------|
| 크리티컬 확률 | 5% | 100% |
| 크리티컬 데미지 | 150% | 300% |

#### 2.2.3 체력 시스템
| 요소 | 상세 |
|------|------|
| 기본 체력 | 캐릭터별 상이 (100 ~ 150) |
| 체력 회복 | 경험치 젬 대량 흡수 시 5% 회복 |
| 무적 시간 | 피격 후 0.5초 |

## 3. 캐릭터 시스템

### 3.1 캐릭터 구조

#### 3.1.1 기본 스탯
| 스탯명 | 설명 | 범위 |
|--------|------|------|
| HP | 체력 | 100 ~ 200 |
| ATK | 공격력 | 10 ~ 30 |
| DEF | 방어력 | 5 ~ 20 |
| SPD | 이동 속도 | 4.5 ~ 6.0 |
| CRIT_RATE | 크리티컬 확률 | 3% ~ 10% |
| CRIT_DMG | 크리티컬 데미지 | 140% ~ 180% |

#### 3.1.2 캐릭터 데이터 구조
```
Character {
    id: string                    // 고유 식별자
    name: string                  // 캐릭터 이름
    rarity: enum(1~6)            // 성급 (1성~6성)

    // 기본 스탯
    baseStats: {
        hp: int
        atk: int
        def: int
        spd: float
        critRate: float
        critDmg: float
    }

    // 성급별 스탯 증가율
    starGrowth: {
        hp: float           // 성급당 HP 증가율 (예: 0.1 = 10%)
        atk: float
        def: float
    }

    // 전용 스킬
    uniqueSkill: {
        id: string
        unlockStar: int     // 해금 성급 (1성)
        upgrade1Star: int   // 1차 강화 성급 (3성)
        upgrade2Star: int   // 2차 강화 성급 (6성)
    }

    // 획득 조건
    unlockCondition: {
        type: enum(DEFAULT, GACHA, EVENT, SHOP)
        value: any
    }
}
```

### 4.4 장비 합성 시스템

#### 4.4.1 기본 합성 (일반 ~ 엘리트)
```
합성 공식: 동일 장비 3개 → 상위 등급 장비 1개

예시:
일반 쿠나이 x3 → 우수 쿠나이 x1
우수 쿠나이 x3 → 레어 쿠나이 x1
레어 쿠나이 x3 → 엘리트 쿠나이 x1
```

#### 4.4.2 에픽 합성
```
합성 공식:
  메인 장비 (엘리트+2) x2 + 재물 장비 (엘리트) x6 → 에픽 장비 x1

필요 총 엘리트 장비: 8개 (메인 2개 + 재물 6개)
필요 총 레어 장비: 8 × 3 = 24개
```

#### 4.4.3 레전드 합성
```
합성 공식:
  메인 장비 (에픽+3) x1 + 동일 장비 (에픽) x2 → 레전드 장비 x1

필요 총 에픽 장비: 3개
필요 총 엘리트 장비: 3 × 8 = 24개
```

#### 4.4.4 합성 플로우차트
```
[일반] ──(x3)──→ [우수] ──(x3)──→ [레어] ──(x3)──→ [엘리트]
```
